Page Description
Example 1 To begin with I have simply consumed the JSON on the server from the API, it is then rendered into a view in play and served to the user.
Example 2 In this version, the API call is cached for up to 5 minutes by Play. This prevents excessive API calls in a short period of time. I have also added show and hide buttons to the page.
Example 3 This example uses AJAX to load the bill into the page.

Assumptions

  • The API hosted on Heroku will always be available.
  • The JSON received by the application will always contain the same structure as the example Json provided for this project. For example, if a bill contains no telephone calls, the calls array property will still be present, just empty.
  • For the first two examples, the number of subscriptions, calls, rentals and buyAndKeeps will always > 0.
  • Ajax calls never fail.
  • Latest version of Firefox or Chrome will be used to browse the site.

Next steps

  • Improve accessibility.
  • Improve support for older browsers.
  • Implement client side and server side failure handling.
  • Support for browsers with disabled Javascript.
  • Use websockets to stream live bill changes to the user in realtime.
  • Learn Angular and reimplement the front end.
